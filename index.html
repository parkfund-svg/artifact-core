HATS 아키텍처: Brain–Artifact–Hand

기존 LLM 자동매매의 두 가지 치명적 문제: 높은 토큰 비용과 느린 실행 속도를 구조적으로 해결합니다.

문제 1

높은 운영 비용

실행마다 LLM 호출로 토큰 비용이 누적됩니다. 거래 횟수가 많을수록 원가가 급증하여 가격 경쟁력이 약화됩니다.

문제 2

느린 실행 속도

수 초의 LLM 응답 지연으로 급변하는 시장에서 슬리피지와 기회 손실이 발생합니다.

해결

HATS: 지능의 분리

  • Brain: 자연어 전략 설계, 1회 LLM 사용
  • Artifact: JSON 실행 설계도 영구 저장
  • Hand: Local ML로 초저지연 실행, Zero-cost

Artifacts로 토큰비용 최저 수렴

LLM 지능을 캡슐화하여 실행 단계에서 재호출 없이 무한 반복 실행이 가능. 토큰 비용 최저, 운영 효율 극대화.

초저지연 실행 엔진

수 ms 단위 저지연으로 매매 신호를 생성하고 주문을 라우팅합니다. 실시간 대응력을 극대화합니다.

1단계 · 전략 지능(Brain)

자연어로 전략을 설명하면 LLM이 이를 분석해 실행 가능한 논리로 설계합니다.

2단계 · Artifact Core

설계된 전략을 경량 JSON Artifact로 변환·저장하여 시스템의 표준 실행 규격으로 사용합니다.

3단계 · 실행 지능(Hand)

Artifact를 입력으로 Local ML이 시세/호가 데이터를 분석해 초고속으로 신호를 생성·실행합니다.

제품·서비스

B2C SaaS와 B2B API로 구성된 포트폴리오. 전략 빌더부터 실거래 엔진까지 단계적으로 제공합니다.

BASIC

향후 5개 캔들의 단기 흐름 예측. 시계열 딥러닝 기반.

PRO

자연어/노드 기반 전략 자동 생성과 백테스트. 차트에 익절/손절·진입선 자동 추가.

자주 묻는 질문

왜 HATS가 비용 효율적인가요?

실행 단계에서 LLM을 호출하지 않습니다. 초기 설계 시 1회만 사용하고, Artifact를 무한 반복 실행하므로 토큰 비용이 발생하지 않습니다.

실행 속도는 어느 정도인가요?

Local ML 최적화로 수 밀리초(ms) 단위의 저지연을 목표로 설계되었습니다. 시장 급변에도 신속하게 대응합니다.

전략 빌더는 어떻게 동작하나요?

자연어로 기술한 매매 규칙을 노드 그래프로 변환하고, Artifact(JSON)로 표준화하여 백테스트 및 실거래에 바로 사용합니다.

B2B API는 어떤 연동을 제공하나요?

시그널 생성, 리스크 관리, 주문 라우팅 엔드포인트를 제공합니다. REST/WebSocket, OAuth2 기반 인증을 지원합니다.
